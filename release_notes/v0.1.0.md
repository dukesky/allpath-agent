# Allpath Agent v0.1.0

Released: 2026-07-15

Allpath Agent v0.1.0 is the first locally runnable MVP. It establishes the
small-agent foundation we can continue learning from and extending: one agent
core, multiple model providers, persistent local state, safe tools,
conversation-first onboarding, and messaging connectors at the edges.

## Highlights

### Local agent MVP

- Install and launch Allpath Agent with the `allpath-agent` command.
- Start in a useful local mode without requiring an API key.
- Persist conversations, resume sessions, search model-routing history, and
  recover cleanly from interruption.
- Keep credentials out of conversation history, workflow state, and runtime
  logs.

### Progressive onboarding

- Connect models and messaging channels through normal conversation rather
  than a mandatory first-run wizard.
- Show one relevant next capability at a time based on prerequisites, prior
  use, cooldowns, and dismissal state.
- Keep onboarding guidance outside the model prompt so it does not invalidate
  prompt caching.

### Multi-model routing

- Connect OpenAI-compatible APIs, Anthropic, Gemini, xAI Grok, OpenRouter,
  Ollama, Claude Code, and OpenAI Codex account authentication.
- Assign models to `fast`, `standard`, and `advanced` roles.
- Route simple work to cheaper eligible models and complex work to the
  advanced profile.
- Inspect active models with `/model` and `/models`, and explain the latest
  routing decision with `/route`.

### Safe tools and durable state

- Use a deterministic tool registry with stable schemas.
- Require explicit approval for side-effecting tools and deny them by default
  in messaging gateways until a channel-safe approval flow exists.
- Store sessions, messages, routing decisions, memory, workflow progress, and
  connector bindings in SQLite.
- Record privacy-safe lifecycle events without prompts, credentials, tool
  arguments, or provider response bodies.

## Messaging Connectors

### Telegram

- Connect through a conversational BotFather walkthrough and hidden Bot Token
  input.
- Verify the bot identity before activation.
- Receive and reply to messages through the shared gateway while preserving a
  stable Allpath session per Telegram conversation.

### Slack

- Connect with official Slack Socket Mode using hidden `xoxb-` Bot and `xapp-`
  App-Level tokens.
- Verify both Slack authentication and WebSocket connection readiness.
- Reply directly in the main conversation for DMs, preserve existing threads,
  and create threads for channel-root messages to reduce channel noise.

### WhatsApp

- Connect through Meta's official WhatsApp Cloud API.
- Verify the Access Token and Phone Number ID before activation.
- Receive signed webhook events at `/webhooks/whatsapp` and validate
  `X-Hub-Signature-256` with the Meta App Secret.
- Send text replies through the official Graph API.
- Run the local webhook on port `8787`; a public HTTPS tunnel is still required
  so Meta can deliver webhook events to a locally running agent.
- Deliberately avoid unofficial QR-code and WhatsApp Web automation approaches.

## Running the Gateway

After connecting at least one messaging channel:

```bash
allpath-agent gateway
```

Telegram, Slack, and WhatsApp can run together in the same gateway process.
Connector conversations reuse persistent local agent sessions across restarts.

For WhatsApp, expose local port `8787` with an HTTPS tunnel and configure the
Meta callback URL as:

```text
https://<public-host>/webhooks/whatsapp
```

Use the same verify token entered during `connect WhatsApp`, then subscribe the
webhook to `messages`.

## Reliability and Validation

- 138 automated tests pass across agent loop, providers, routing, tools,
  storage, onboarding, interruption handling, installation, and connectors.
- CI validates Python 3.11, 3.12, and 3.13.
- The local installer is idempotent and supports source-linked development
  installs for rapid testing.

## Current Limitations

- The gateway must remain running to receive messages.
- WhatsApp requires a Meta Business app and a public HTTPS webhook endpoint.
- WhatsApp currently supports inbound and outbound text messages; attachments,
  templates, reactions, and interactive messages are not implemented.
- Proactive WhatsApp conversations may require an approved message template
  under Meta's platform policies.
- Messaging-channel side-effect approvals are intentionally denied until a
  safe remote approval experience is implemented.
- Cron jobs, additional channel connectors, browser automation, subagents, and
  a desktop UI remain future milestones.

## Upgrade

Existing source-linked local installations load this release on the next
`allpath-agent` process start. For a fresh local installation, follow the
one-line installer instructions in the project README.
